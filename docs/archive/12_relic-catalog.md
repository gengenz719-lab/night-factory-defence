# レリック設計集(カタログ v2) — 名前・効果まで確定した叩き台

> 実装前のデザイン。数値・採否はレビューで調整する。OK後に NfdRelicData を拡張して実装。
> 着想元: **Vampire Survivors**(進化・パッシブ積み)/ **Brotato**(タグ・セット・リスク付き)/ **Slay the Spire**(レアリティ・シナジー・アーキタイプ)。
> 関連: [07_ideas.md](../07_ideas.md) / [00_game-constitution.md](00_game-constitution.md)

## 1. 設計思想

- **積む楽しさ**(VS): 同じ系統を重ねるとどんどん尖る。`stackable` で表現
- **タグとセット**(Brotato): 各レリックに **系統タグ** を付け、同系統が揃うと強い。ビルドの軸になる
- **進化/シナジー**(VS/StS): 特定の2枚が揃うと **進化**(上位効果に化ける)。「揃った!」の瞬間を作る
- **リスク付き**(Brotato): デメリット付きの強レリックで、尖った賭けができる
- **アーキタイプ**(StS): 下の「ビルド原型」に沿ってレリックを配置。どれを引いても何かの軸に噛む

## 2. 系統タグ

| タグ | 意味 |
|---|---|
| 🔫 銃 | プレイヤーの攻撃 |
| 🏰 拠点 | タレット・壁 |
| 🏭 工場 | 採掘・加工・経済 |
| 🏃 遊撃 | 移動・HP・生存 |
| 🎲 賭博 | デメリット付きハイリスク |

## 3. ビルド原型(特定ビルドを組む楽しさ)

引いたレリックがどれかの軸に噛むよう設計。**5原型**。

| 原型 | ファンタジー | 主なレリック | 進化の狙い |
|---|---|---|---|
| **弾幕ガンナー** 🔫 | 自分で薙ぎ倒す。工場は最小 | 速射/増し弾/跳弾/掃射/弾道学 | 速射+跳弾=「弾幕」で全貫通 |
| **自動要塞** 🏰 | タレットに任せ自分は遊撃 | 砲術/連射砲/二連装/増設架台/オーバードライブ | 砲術+連射砲=「管制」で2体同時狙撃 |
| **工場長** 🏭 | 資源過剰→物量で押す | 血の採掘/効率炉/夜勤/ゾンビ燃料 | 血の採掘+効率炉=「自動化」 |
| **不死身遊撃** 🏃 | 前線でHP管理しつつ暴れる | 硬い体/背水/吸血装置/応急キット | 硬い体+背水=「火事場」 |
| **賭博師** 🎲 | ハイリスク超火力 | 血の代償/一点賭け/暴走炉 | デメリットを他で埋めて成立させる |

## 5. シナジー / 進化(揃うと化ける)

所持している2枚が揃うと自動で進化効果が乗る(VSの合成に相当)。第1弾は太字の3つを推奨。

| 組合せ | 進化名 | 効果 |
|---|---|---|
| **速射 + 跳弾** | **弾幕** | 貫通が無限化(貫通してもダメージ減衰なし) |
| **砲術 + 連射砲** | **管制システム** | タレットが射程内の2体を同時狙撃 |
| **血の採掘 + 効率炉** | **自動化ライン** | 鉄が上限なら自動で弾薬に変換 |
| 炎上弾 + 掃射 | 火炎放射 | 扇状攻撃が短時間の炎上フィールドを残す |
| 硬い体 + 背水 | 火事場 | 低HP時に移動速度も+30% |
| ゾンビ燃料 + 夜勤 | 前線工場 | 夜の敵ドロップ量+100% |
| 強化壁 + 自己修復壁 | 城塞 | 壁が夜でも徐々に回復 |
| 血の代償 + 吸血装置 | 修羅 | 撃破時のHP回復が3倍 |

## 7. バランスの考え方

- **スタック上限**: 同一 +% 系は **3回まで**(青天井インフレ防止)
- **リスク補正**: 🎲賭博は効果を強めにし、他レリックで穴を埋める前提。単体では博打
- **経済の天秤**: 工場系を強くしすぎると「置くだけ」化する。工場ビルドでも夜は忙しい設計を保つ
- **出現重み**(Wave帯 × レアリティ)は前版 §3 を踏襲:

| Wave帯 | Ⅰ | Ⅱ | Ⅲ | ★ |
|---|---|---|---|---|
| 1〜3 | 70 | 28 | 2 | 0 |
| 4〜6 | 45 | 40 | 14 | 1 |
| 7〜9 | 20 | 40 | 33 | 7 |

- **節目保証**(任意): Wave5クリアで1枚Ⅲ以上、Wave9クリアで1枚★確定
- **3択**: うち最低1枚は「今のビルドに噛む系統」を優先して出す案も(引きの弱さを緩和)

## 8. 実装メモ(OK後)

- `NfdRelicData` に追加: `rarity`(Ⅰ/Ⅱ/Ⅲ/L)、`tag`(系統)、`stackable`、`jobOnly`(ジョブID or なし)、`synergyKey`
- 出現重みテーブルを SO 化 → `OfferRelics` を重み抽選+ジョブ/ビルド傾斜に変更
- effectType を拡張(炎上/クリティカル/貫通無限/連鎖/ドロップ資源/シールド/背水/自己修復 など)
- シナジーは RelicManager が「所持セット」を判定して進化効果を有効化
- 第1弾スコープ案: **レアリティ+出現重み+既存再配置+新規コモン/レア数種+進化3つ**。エピック以降とジョブ専用は後続
