# AIバイブコーディングの回し方

AIを「3人目の開発メンバー」として扱う。役割は4つ。

| 役割 | 使い方 |
|---|---|
| 先生 | 分からない用語・エラーを聞く |
| 相棒 | 小さい機能を書いてもらう |
| レビュアー | 書いた(書かせた)コードを読ませて危ない所を指摘させる |
| 企画係 | レリック案・敵案・バランス調整案を量産させる |

## 大原則

```text
AIには「小さい作業」を渡す。
「このゲームを作って」ではなく「この1機能を、このファイルに、この条件で」。
```

このリポジトリには CLAUDE.md にAI向けルールが書いてあるので、
Claude Code / Cursor をリポジトリのフォルダで開けば、AIが自動的にプロジェクトの流儀を守る。

## プロンプト例(コピペして使う)

### 機能を作ってもらう

```text
prototype/js/buildings.js に「昼フェーズ中、壁を右クリック長押しで修理する」機能を足したい。
- 修理コストは鉄。数値は config.js に定義して
- 既存のコードの書き方に合わせて、日本語コメント多めで
- 変更したファイルと、動作確認の手順を最後にまとめて
```

### エラーを直してもらう

```text
ブラウザでゲームを開いたらエラーが出た。

エラー文(F12のConsoleからコピペ):
(ここに貼る)

直前にやったこと:
(例: config.js に新しいレリックを追加した)

原因、直し方、再発防止を初心者にも分かるように教えて。
```

### バランス調整案を出させる

```text
docs/01_prototype-spec.md と prototype/js/config.js を読んで。
Wave 7 が急に難しすぎる。敵予算表とスポーン間隔の調整案を3パターン、
それぞれの狙い(プレイヤーに何を要求する難しさか)つきで提案して。
```

### 週次クエストを作らせる

```text
docs/03_roadmap.md を読んで。
初心者メンバーが2時間で達成できる今週の開発クエストを5つ、難易度★1〜5つきで出して。
```

### レビューさせる

```text
今回の変更差分をレビューして。
特に「将来マルチプレイ化するときに困る書き方」と
「config.js に置くべき数値の直書き」がないかを見て。
```

## AIとの作業で守ること

1. AIが書いたコードは必ず一度読む。読めなければ「1行ずつ解説して」と頼む
2. 1回の依頼は1機能。大きな依頼は「まずタスクを分解して」から
3. 良かったプロンプトはDiscordの #aiに聞いたこと に貼って共有する
4. AIの提案がゲームの憲法(docs/00)に反するときは断る
