## 1. 設計の狙い

- **段階(レアリティ)で「引きの重み」を作る** … 序盤は無難に強くなり、終盤に化ける
- **ビルドの分岐** … プレイヤー強化/拠点強化/工場強化のどれに寄せるかで別の遊びになる
- **シナジー** … 2枚の組合せで追加効果 →「あの2枚が揃った!」の瞬間を作る
- 憲法どおり「防衛を楽にする小さな工場」から外れる強化は入れない

## 2. レアリティ4段階

| 段階 | 位置づけ | 効果の目安 | 1ランでの出やすさ |
|---|---|---|---|
| Ⅰ コモン | 無難・確実 | +15〜25% / 即時小 | 多い(序盤の主力) |
| Ⅱ レア | しっかり強化・条件付き | +30〜50% / 小さな新メカ | 中盤の主力 |
| Ⅲ エピック | ビルドを決める | +60〜100% / 新メカ | 終盤で増える |
| ★ レジェンダリ | ルールを変える | 別次元(1ランで0〜2枚) | 稀。節目で保証枠 |

## 3. 出現ロジック(案)

- Waveクリアごとに **3択**(既存どおり)。未所持から抽選。
- **Wave帯でレアリティの重みが変わる**(数字は重み。合計で正規化):

| Wave帯 | Ⅰ | Ⅱ | Ⅲ | ★ |
|---|---|---|---|---|
| 1〜3 | 70 | 28 | 2 | 0 |
| 4〜6 | 45 | 40 | 14 | 1 |
| 7〜9 | 20 | 40 | 33 | 7 |

- **節目保証**: Wave5クリアは1枚をⅢ以上、Wave9クリアは1枚を★確定 … など「山場のご褒美」を作る(採否は要検討)
- **スタック可否**: レリックごとに `stackable` を持たせ、重ねられるもの(+%系)は再抽選対象に残す。ユニーク(レジェンダリ等)は1回のみ
- **ジョブ連動(将来)**: エンジニアはタレット系の重み+、通信兵は火力系+ … ジョブでプールを傾ける(ジョブ実装後)

## 6. シナジー表(案)

所持している2枚が揃うと追加効果。

| 組合せ | シナジー名 | 追加効果 |
|---|---|---|
| 跳弾 + 速射 | 弾幕 | 貫通してもダメージが減衰しない |
| 砲術 + 連射砲 | 管制 | タレットが射程内の2体を同時狙撃 |
| 血の採掘 + 効率炉 | 自動化 | 鉄が上限なら自動で弾薬に変換 |
| 硬い体 + 背水 | 火事場 | 低HP時に移動速度も上昇 |
| 強化壁 + 自己修復壁 | 城塞 | 壁が夜でも徐々に回復 |
| ゾンビ燃料 + 夜勤 | 前線工場 | 夜の敵ドロップが増える |

## 7. 実装メモ(OKが出たら)

- `NfdRelicData` に追加: `rarity`(enum Ⅰ/Ⅱ/Ⅲ/L)、`stackable`(bool)、必要なら `synergyKey`
- 出現重みテーブルを SO 化(Wave帯×レアリティ)。`OfferRelics` を重み抽選に変更
- 新レリックの effectType を追加(背水/炎上/自己修復/掃射/ドロップ資源 など。既存の enum を拡張)
- シナジーは RelicManager が「所持セット」を見て追加効果を有効化する形
- スタック時の上限(例: 同一+%は3回まで)を決める

## 8. 決めてほしいこと(レビュー観点)

1. **レアリティの効果幅**(§2の目安)でよいか

4. **追加案**(§5)から採用するもの/落とすもの
5. **シナジー**(§6)は第1弾で入れるか、後回しか
