# パートナー向け参加ガイド

プログラミング経験がなくても参加できます。AIが3人目の開発メンバーです。

## 必要なもの(全部無料)

1. ブラウザ(遊ぶだけならこれだけ)
2. [VS Code](https://code.visualstudio.com/)(開発に参加するなら)
3. [Git](https://git-scm.com/)(履歴管理。インストールだけでOK、コマンドは覚えなくていい)
4. AIアシスタント — Claude Code / Cursor / GitHub Copilot のどれか
5. Discord(相談・成果報告)

## 参加手順

```text
1. このフォルダ(リポジトリ)をもらう or git clone する
2. prototype/index.html をブラウザで開いて、まず遊ぶ
3. docs/00_game-constitution.md を読む(5分)
4. Discordの #今日やること で、やりたいクエストを宣言する
5. AIに手伝ってもらいながら作る(→ docs/06_ai-collab-guide.md)
6. 動いたら #できた報告 にスクショか動画を貼る
```

## 壊すのが怖い人へ

- Gitで全履歴が残っているので、**何をどう壊しても必ず元に戻せます**
- 変更する前に最新版を取得(pull)する、だけ守ればOK
- 分からなくなったらAIに「gitで1つ前の状態に戻したい」と聞けば手順を教えてくれます

## 共同開発ルール(7か条)

```text
1. 同じファイルを同時に触らない(Discordで宣言してから作業)
2. 数値調整だけなら js/config.js だけ触ればいい(壊れにくい・効果がすぐ見える)
3. 変更前に必ず最新版を取得する
4. 作業後は「何を変えたか」をDiscordに書く
5. AIが書いたコードは必ず人間が一度読む(読んで分からなかったらAIに解説させる)
6. 動かない状態で共有しない
7. 毎週1回、最新版をみんなで遊ぶ
```

## Discordチャンネル構成(推奨)

```text
#雑談
#今日やること   … 作業宣言(ファイル衝突防止を兼ねる)
#できた報告     … スクショ・動画。いちばん大事なチャンネル
#バグ報告       … バグは敵キャラ扱いで楽しく(下記フォーマット)
#アイデア置き場
#aiに聞いたこと … 良かったプロンプトの共有
```

### バグ報告フォーマット

```text
バグ名: 透明ゾンビ
症状: ゾンビが見えないのに拠点が壊れる
危険度: ★★★
発生条件: Wave3でたまに起きる
討伐担当: (空欄でOK、やりたい人が名乗る)
```

## 最初のおすすめクエスト(経験ゼロ向け)

```text
★ js/config.js を開いて、ゾンビの速度や壁のHPを変えて遊ぶ
★ レリックの名前と説明文を1個考えて追加する(AIに「このファイルのRELICSに追加して」と頼む)
★ 遊んでいて気づいたことを #バグ報告 / #アイデア置き場 に書く
```
