# Godot版 ルート・UI・受け入れ仕様 v0.1

> 関連: [13_godot-redevelopment-spec.md](13_godot-redevelopment-spec.md) / [14_godot-technical-spec.md](14_godot-technical-spec.md)

## 1. ルートマップ仕様

### 1.1 10列構造

| 列 | 対応Wave | 地域 | 制約 |
|---:|---:|---|---|
| 1 | 1 | 崩壊郊外 | 道路固定 |
| 2 | 2 | 崩壊郊外 | 道路/廃品/救難 |
| 3 | 3 | 崩壊郊外 | ガレージを最低1候補 |
| 4 | 4 | 崩壊郊外 | エリート最大1候補 |
| 5 | 5 | 地域境界 | 中ボス固定 |
| 6 | 6 | 汚染工業地帯 | 道路/廃品/異常 |
| 7 | 7 | 汚染工業地帯 | ガレージを最低1候補 |
| 8 | 8 | 汚染工業地帯 | エリートを最低1候補 |
| 9 | 9 | 汚染工業地帯 | エリート/異常/救難 |
| 10 | 10 | 群体巣道 | 最終ボス固定 |

### 1.2 ノード生成

- 列1、5、10は1ノード。その他は2〜3ノード。
- 各ノードは次列の1〜2ノードへ接続する。
- 全ノードに入口と出口が最低1本ある。
- 列を飛び越す辺、後戻りする辺、同列内の辺は作らない。
- プレイヤーの現在ノードから次列に最低2候補を保証する。ただしボス直前は1候補でもよい。
- 3列連続で同じ種類だけを選ばせない。
- 同じ列にエリートを2つ置かない。

### 1.3 生成例

```text
W1       W2               W3               W4          W5
[道路] ─ [廃品] ──────── [ガレージ] ───── [道路] ─┐
   └──── [道路] ─┬────── [救難] ───────── [エリート] ├─ [ハルク]
                 └────── [道路] ───────── [廃品] ───┘

W6             W7               W8               W9          W10
[異常] ────── [ガレージ] ───── [エリート] ───── [異常] ─┐
[道路] ─┬──── [道路] ───────── [救難] ───────── [エリート] ├─ [コロニー]
[廃品] ─┘       └──────────── [道路] ───────── [救難] ───┘
```

### 1.4 ノード情報カード

選択時に必ず表示する情報:

- ノード名とアイコン
- Wave番号、推定脅威（低/標準/高/極高）
- 新登場または主力となる敵アイコン最大3つ
- 危険イベント
- 基本報酬倍率
- 特別報酬またはサービス
- 消費燃料。通常1、遠回り2
- 接続先の次ノード

未知情報は`?`で隠すが、「知らないこと自体」が選択後に不利と感じられないよう、危険度は必ず公開する。

## 2. ノード報酬

| ノード | 追加報酬/サービス |
|---|---|
| 道路 | 追加なし。最も安定 |
| 廃品回収地 | 車外回収物3〜5個、モジュール購入候補2つ |
| ガレージ | 修理コスト-30%、モジュール購入候補4つ、売却額+10% |
| 救難信号 | 任意目標成功で全員HP全回復または再抽選+1 |
| 異常地帯 | 特殊系レリック重み×2、危険ルール1つ |
| エリート | スクラップ/補給1.5倍、レア以上保証 |
| ボス | フレーム拡張または地域報酬、レアリティ保証 |

- 共有レリック3択はWave 1〜9のノードで発生する。Wave 10の最終ボス後はリザルトへ進む。
- 追加モジュール候補は購入権であり、無料取得ではない。
- ガレージ以外でも既所有モジュールの移動・撤去は可能。

## 3. イベントカタログ

初期版は以下8種類。長文アドベンチャーにはせず、1画面・2〜3択で終える。

| ID | イベント | 選択例 |
|---|---|---|
| event_abandoned_bus | 放棄されたバス | 燃料を使って捜索/通過 |
| event_survivor | 取り残された生存者 | 補給を渡して再抽選/断る |
| event_roadblock | 封鎖道路 | スクラップで除去/危険な迂回 |
| event_black_market | 移動商人 | モジュール購入/レリック交換/去る |
| event_strange_signal | 奇妙な信号 | 特殊レリック候補/車体ダメージを避ける |
| event_fuel_tanker | 燃料車 | 燃料+2/敵増援を避けて+1 |
| event_quiet_night | 静かな区間 | 全修理/次Wave報酬+25%を選択 |
| event_trophy | 思い出の品 | 装飾解放/スクラップ化 |

- イベント結果はホストが確定する。
- マルチは選択肢を投票。15秒締切。
- 取り返しのつかない恒久ペナルティをランダムで強制しない。

## 4. 初回チュートリアル

チュートリアル専用の別ゲームを作らず、初回ランのWave 1〜3へ段階的に組み込む。

### Wave 1前

1. 左右移動、ジャンプ、はしご。
2. 射撃窓へ移動し、照準と射撃。
3. 損傷した練習パネルを修理。
4. 準備完了。

必須入力を行うまでWaveを開始しない。マルチでは各自の完了状態を表示する。

### Wave 1

- 前方からウォーカーだけが出る。
- 最初の敵で射撃ヒント。
- 外装が90%未満になったら修理ヒント。
- 残り30秒で後方攻撃を1群だけ出し、車内移動を教える。

### Wave 1後

- レリック投票を説明。
- 3枚の系統色と「共有効果」であることを強調。
- ルート選択を説明。

### Wave 2

- ランナーと回避を説明。
- 終了後、個人スキルは各自の効果であることを説明。

### Wave 3

- クライマー1体を確定出現。
- 屋根への移動と侵入口の応急封鎖を説明。

### 再表示

- 設定からチュートリアルヒントをリセットできる。
- 既読ヒントはプロフィールへ保存し、通常は再表示しない。
- 新システムの初回だけ短いヒントを出す。

## 5. HUDワイヤーフレーム

```text
┌──────────────────────────────────────────────────────────────────────────┐
│ P1 HP/ROLE                 WAVE 4   01:37          HULL / POWER / FUEL    │
│ P2 HP/ROLE                   危険: 群れ横断       FRONT ROOF REAR LOWER  │
│ P3 HP/ROLE                                                               │
│ P4 HP/ROLE                                                               │
│                                                                          │
│                    ┌────────────────────────────┐                        │
│      enemies →     │       車両断面・車内       │      ← enemies         │
│                    │ players / modules / breach │                        │
│                    └────────────────────────────┘                        │
│                                                                          │
│ WEAPON / AMMO /     [E 修理  42%] [短い通知]      距離 ──●──── 任意目標  │
│ DODGE / ABILITY                                                           │
└──────────────────────────────────────────────────────────────────────────┘
```

### 5.1 表示優先度

常時表示:

- Wave、時間
- 自分と味方のHP/ダウン
- 車体と外装危険状態
- 自分の武器、弾倉、回避、固有能力
- 現在のインタラクト

条件表示:

- 電力不足
- 襲撃方向
- 危険イベント
- 任意目標
- ピン、短い通知

メニュー中のみ:

- 詳細ステータス
- レリック一覧
- 車両電力内訳
- 全モジュールHP

## 6. HUD警告ルール

| 状態 | 表示 | 音 |
|---|---|---|
| 車体50%未満 | 右上を黄、1回通知 | 中警告1回 |
| 車体25%未満 | 右上を赤、緩い点滅 | 低頻度警告 |
| 外装破壊 | 対応方向を赤、侵入口矢印 | 方向付き破壊音 |
| 電力不足 | 停止設備と電力値を黄 | 短い電源低下音 |
| 味方ダウン | 左上対象を赤、距離矢印 | ボイス/警告1回 |
| 大攻撃 | 予告形状+方向 | 固有予告音 |

- 常時鳴る低HPビープ音は採用しない。
- 同じ警告を1秒以内に重ねない。
- 画面振動、フラッシュ、ダメージ数値は設定で個別に弱められる。

## 7. 画面仕様

### 7.1 ロビー

- ルームコード/招待状態
- P1〜P4の参加枠
- キャラクター、武器、衣装
- ホストの難易度、シード設定
- 全員の準備完了
- ホストだけが開始可能

### 7.2 レリック選択

- 3カードを横並び。
- 系統、レアリティ、名前、1文効果、現在スタック、シナジー候補を表示。
- 各カードの投票者色を表示。
- 20秒タイマー、再抽選ボタン、所持レリック一覧。
- 効果文は最大3行。複雑な計算式は詳細表示へ。

### 7.3 改装

- 左: 購入/所持モジュール一覧。
- 中央: 車両グリッド。
- 右: 電力、資源、選択モジュール詳細。
- 配置可能セルは青、不可能は赤、通路違反は黄色の経路表示。
- 確定前の差分と払い戻し額を常時表示。
- マルチで他人が持っているモジュールは色枠と名前で表示。

### 7.4 リザルト

- 勝敗、到達Wave、時間。
- 車両構成の最終断面スナップショット。
- 取得レリックと進化。
- 各プレイヤーの与ダメージ、修理量、蘇生、設備稼働、回収量。
- MVP順位付けはしない。役割の違う貢献を並列表示する。
- マスタリーXP、設計図、装飾解放。

## 8. 設定とアクセシビリティ

初期アルファから必要:

- マスター/BGM/SE/環境音の個別音量
- 全入力の再割当
- ゲームパッド振動
- 画面振動0〜100%
- ダメージフラッシュ0〜100%
- 照準補助0〜100%（ゲームパッドのみ）
- UIスケール80〜140%
- 色覚補助パレット3種
- 字幕/重要音インジケーター
- 押し続け操作とトグル操作の切替
- ダメージ数値の表示/非表示

### 8.1 難易度とアクセシビリティの分離

- 敵予算やダメージを変える項目は難易度設定。
- UI、色、入力、揺れ、照準補助はアクセシビリティ設定。
- アクセシビリティ設定で報酬を減らさない。

## 9. エラーと切断UI

| 状況 | 挙動 |
|---|---|
| クライアント接続失敗 | 理由、再試行、戻るを表示 |
| 戦闘中切断 | 画面上部に再接続中。ゲーム状態はホスト側で退避 |
| 90秒以内再接続 | 最新スナップショット受信後に操作を戻す |
| ホスト切断 | 中断保存を試み、セッション終了理由を表示 |
| バージョン不一致 | ホスト/参加者のバージョンを表示し接続拒否 |
| セーブ破損 | 破損データ退避、復旧できなかった項目を説明 |

- 「不明なエラー」だけで終わらせず、ログ識別IDを表示する。
- 接続エラーでプロフィール進行を消失させない。

## 10. 最小スライス受け入れテスト

### A. 起動とロビー

- [ ] Windowsビルドがエラーなく起動する。
- [ ] ホスト作成と1クライアント参加ができる。
- [ ] ガンナー/エンジニアを選び、両者準備完了で開始する。
- [ ] バージョン不一致接続を拒否する。

### B. 車内操作

- [ ] 8×4、2階層の車内で左右移動、ジャンプ、床降り、はしご、屋根移動ができる。
- [ ] 2プレイヤーが互いに衝突せずすれ違える。
- [ ] 車両から取り残されると警告後に牽引される。
- [ ] モジュール配置で通路とはしごを完全に塞げない。

### C. 戦闘

- [ ] ウォーカー、ランナー、クライマーが定義Waveで出現する。
- [ ] プレイヤー射撃の見た目は即時、ダメージはホスト確定になる。
- [ ] 外装破壊で侵入口ができ、クライマーが車内へ入る。
- [ ] 回避無敵、ダウン、蘇生、離脱後復帰が仕様値どおり動く。
- [ ] 車体HP0で全端末が敗北画面へ移る。

### D. 車両

- [ ] 発電機、射撃窓、タレット、工作台を配置できる。
- [ ] 電力不足で優先度の低い設備が停止する。
- [ ] 戦闘中修理は60%上限、Wave間は100%まで修理できる。
- [ ] タレットが熱100で停止し、冷却後に復帰する。

### E. Waveと報酬

- [ ] 90秒経過後、新規スポーンが止まり10秒離脱へ移る。
- [ ] 残敵がいても車両生存でWaveクリアになる。
- [ ] 同じ3枚のレリック候補が全端末に表示される。
- [ ] 投票の多数決/同票ランダムがホスト確定で一致する。
- [ ] 2列のルート候補から投票し次Waveへ進める。

### F. 通信

- [ ] 150ms RTT、2%損失で10分間進行できる。
- [ ] 背景、粒子、薬莢を同期せず見た目が成立する。
- [ ] クライアント切断後90秒以内に同状態で復帰する。
- [ ] 共有資源、車両HP、レリック、ルートに不一致がない。

### G. UIと入力

- [ ] 1280×720で重要UIが切れず、車両中央を隠さない。
- [ ] ゲームパッドだけでロビーから仮勝利まで操作できる。
- [ ] キー再割当が保存される。
- [ ] UIスケール140%でも主要操作が可能。

## 11. フルラン・アルファ受け入れ

- [ ] 全10Wave、ボス2種、エンディングまで進行する。
- [ ] 敵10種が段階的に紹介される。
- [ ] キャラクター4種×スキル12種がデータ定義どおり動く。
- [ ] モジュール12種を配置・強化・売却できる。
- [ ] レリック24種、レアリティ、保証、スタック、進化3組が動く。
- [ ] マップ生成が10列、接続、ボス収束、ノード制約を守る。
- [ ] 1〜4人の人数倍率を適用できる。
- [ ] ソロ中断保存とマルチホスト中断保存を再開できる。
- [ ] 90秒再接続がWave間と戦闘中の両方で機能する。
- [ ] リザルトで各貢献値とアンロックを表示する。
- [ ] 1920×1080で目標60fps、負荷上限でも最低30fpsを維持する。
- [ ] 致命的エラー、進行不能、セーブ破損がない状態で3ラン連続完走する。

## 12. 開発完了の定義との違い

- 最小スライス合格は「このゲームの核が技術的・遊びとして成立する」状態。
- フルラン・アルファ合格は「仕様の全要素を最初から最後まで検証できる」状態。
- 製品完成にはアート、音、公開マッチ、Steam、実績、最終バランス、QA、ストア対応が別途必要。
- 本仕様の目標は、製品完成ではなく迷いなくGodot開発を開始できる状態まで確定すること。
