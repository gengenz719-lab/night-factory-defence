# Unity版 開発マイルストーン(タスク分解)

「システム(状態・ロジック)を先に作り、各機能に最小限の情報UIを同時に付ける」方針のタスク一覧。
1タスク = AIとの1セッション(30分〜2時間)を目安に分解してある。上から順にやる。

関連: [10_unity-migration.md](10_unity-migration.md)(移植計画の全体像)、[01_prototype-spec.md](01_prototype-spec.md)(仕様のリファレンス)

## 方針: なぜシステム先行か

1. **面白さはHTML5プロトタイプで証明済み**。Unity版の仕事は「同じ遊びを再現する」ことで、見た目は後から差し替えられる
2. **UIは状態を読むだけ**(CLAUDE.mdの分離ルール)。先に状態がないとUIはモック(嘘の画面)になり、後で作り直しになる
3. ただしこのゲームのHUDは飾りではなく**ゲームプレイの一部**(状況判断の速さが命)。だから「アートの磨き込み」は後回しでも、「情報のUI」は機能と同時に最小実装する

```text
機能を1つ作る → その機能の数字が読める最小HUDを付ける → 遊んで確認 → コミット
```

UIイメージ(codex作成のモック)は**最終目標**として docs に置き、毎タスクの「最小HUD」をその配置(左上=コア/フェーズ、上中央=Wave、左下=資源、下中央=建設、右下=ミニマップ)に寄せていく。

| # | タスク | 完了条件 |

## 進め方のルール(再確認)

- 1タスク終わるごとに: 遊べる状態を確認 → コミット → #できた報告
- 数値は必ずScriptableObjectに(ロジック直書き禁止)
- HTML5プロトタイプは**仕様のリファレンス**として凍結(バランスの真実はE1完了までconfig.js側)
