# Unity版 開発マイルストーン(タスク分解)

「システム(状態・ロジック)を先に作り、各機能に最小限の情報UIを同時に付ける」方針のタスク一覧。
**1タスク = AIとの1セッション**(30分〜2時間)を目安に分解してある。上から順にやる。

関連: [10_unity-migration.md](10_unity-migration.md)(移植計画の全体像)、[01_prototype-spec.md](01_prototype-spec.md)(仕様のリファレンス)、`prototype/js/config.js`(数値の正)

## 方針: なぜシステム先行か

1. **面白さはHTML5プロトタイプで証明済み**。Unity版の仕事は「同じ遊びを再現する」ことで、見た目は後から差し替えられる
2. **UIは状態を読むだけ**(CLAUDE.mdの分離ルール)。先に状態がないとUIはモック(嘘の画面)になり、後で作り直しになる
3. ただしこのゲームのHUDは飾りではなく**ゲームプレイの一部**(状況判断の速さが命)。「アートの磨き込み」は後回しでも、「情報のUI」は機能と同時に最小実装する

```text
機能を1つ作る → その機能の数字が読める最小HUDを付ける → 遊んで確認 → コミット
```

## 単位変換ルール(プロトタイプ→Unity)

プロトタイプは 1タイル=40px、Unityは 1タイル=1unit。**px値は÷40する**。

| 項目 | config.js | Unity値 |
|---|---|---|
| ウォーカー速度 | 42 px/s | 1.05 u/s |
| ランナー速度 | 95 px/s | 2.375 u/s |
| タンク速度 | 26 px/s | 0.65 u/s |
| プレイヤー速度 | 180 px/s | 4.5 u/s |
| タレット射程 | 170 px | 4.25 u |
| 弾速(プレイヤー/タレット) | 520 / 480 px/s | 13 / 12 u/s |
| アグロ距離 | 110 px | 2.75 u |

## Phase A: ゲーム状態の土台

| # | タスク | 具体的にやること | 完了条件 |
|---|---|---|---|
| A1 | ✅ 状態/頭脳/描画の分離 | `NfdGameState`(純データ) + `NfdGameManager`(状態変更の唯一の入口) + `NfdSliceHud`(読むだけ) | 縦スライスがGameState経由で動く。検証済み |
| A2 | ScriptableObject化 | `NfdEnemyData`(hp/speed/cost/minWave/buildingDps/playerDmg/プレハブ参照)、`NfdBuildingData`派生4種(壁/タレット/採掘機/加工炉)、`NfdWaveData`(予算10本 `[10,15,22,30,42,38,52,68,85,120]`・昼45秒・報酬・初期資源80/80・スポーン間隔0.9)、`NfdPlayerData`、`NfdRelicData`。config.jsの数値を上の換算表で移す。アセットはエディタスクリプトで自動生成 | インスペクタで数値を変えると挙動が変わる(例: walker速度) |
| A3a | 昼夜フェーズマシン | GameStateに `Phase(Day/Night)` と `PhaseTimer` を追加。昼45秒(Spaceでスキップ)→夜→敵全滅で次の昼→Wave番号+1。10Wave目クリアで勝利 | 昼夜が10回まわって勝利画面が出る |
| A3b | Wave予算スポーン | enemies.jsの予算編成を移植: Wave予算からcost引きで敵種を抽選(minWave考慮)、1回に1〜3体グループ、間隔0.9秒。Wave10はタンク4体確定から開始。敵3種(ランナー/タンク)のプレハブとSDFスプライトを追加 | Wave1はウォーカーのみ、Wave4からタンク混入、Wave10が山場になる |
| A4 | 最小HUD更新 | 左上: フェーズ・残り時間・コアHPバー / 上中央: Wave n/10・残敵数。モックの配置に合わせる(OnGUIのままでよい) | 一目で「今なにをすべきか」が分かる |

## Phase B: 建設

| # | タスク | 具体的にやること | 完了条件 |
|---|---|---|---|
| B1 | グリッド設置システム | `NfdBuildGrid`(30×17の占有マップ、タイル⇔ワールド変換)、マウス追従ゴースト(置ける=白/置けない=赤)、設置/撤去のAPIをGameManagerに。鉱床・コア・既存建物のタイルは設置不可 | ゴーストが出て、クリックで仮建物が置ける |
| B2a | 壁の設置と撤去 | WallData(鉄5・HP120)。鉄を消費して設置、右クリックで撤去して50%返金。建物HPと破壊処理 | 鉄が減って壁が建ち、撤去で鉄が半分戻る |
| B2b | 敵が建物を攻撃する | enemies.js移植: コアへの進路上に建物があれば立ち止まって攻撃(buildingDps)。壁破壊で進軍再開 | 壁で敵の流れを堰き止められ、放置すると壊される |
| B3 | タレット | TurretData(鉄25・HP80・射程4.25・毎秒2発・ダメージ12)。範囲内の最寄り敵を自動射撃、**共有倉庫の弾薬を1発1消費**、弾薬0で停止。既存Bulletを流用 | 弾薬が切れるとタレットが黙る(工場の動機付け) |
| B4 | 建設ホットバーUI | 下中央に4スロット([1]〜[4]キー)。アイコン・コスト表示・選択中ハイライト・資源不足はグレー表示 | モックのBuildバーと同じ操作感 |
