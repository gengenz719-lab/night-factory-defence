## 週次クエスト方式(旧版から継続)

毎週テーマを決めて、週末に30分遊ぶ。

```text
30分作業 → 30分テスト → 30分笑ったバグ整理 → 30分次のクエスト決め
```

### 直近のクエスト候補(難易度つき)

| クエスト | 難易度 | 内容 |
|---|---|---|
| バランスの神になる | ★☆☆☆☆ | 敵HP・車体HP・Wave時間の数値だけ触って2Waveの体感を調整 |
| パートナーテスト第1回 | ★☆☆☆☆ | Web版リンクを渡して遊んでもらい、感想を [07_ideas.md](07_ideas.md) へ記録 |
| 敵バリエーション | ★★☆☆☆ | 敵1種を仕様の初期ロスター([16 §4](16_combat-vehicle-balance.md))から追加 |
| 走行感アップ | ★★☆☆☆ | 背景レイヤーを1枚追加/天候・時間帯の色味変化 |
| 装飾はじめました | ★★★☆☆ | 車内に飾りを置けるだけの最小実装(機能なし・見た目だけ) |
| ルートの骨 | ★★★☆☆ | Wave間に「2択のルート選択画面」を仮UIで挟む |
| モジュール第1号 | ★★★★☆ | 発電機 or 射撃窓を配置できるようにする([16 §7](16_combat-vehicle-balance.md)) |

## Steamに向けて(旧版の方針を継続)

```text
完成してから売るのではなく、
ストアページ公開 → デモ公開 → ウィッシュリスト獲得 → Steam Next Fest
の順で早めに露出を始める。

デモの合格ライン:
- 10〜20分で1ラン終わる(仕様は25〜35分ランなので、デモは短縮構成)
- 2人で遊ぶと面白い
- スクショで何のゲームかわかる

注意: AI生成アセットを使う場合、Steamは申請時にAI利用の開示が必須。
制作記録(godot/assets/art/README.md)を残し続けること。
```
