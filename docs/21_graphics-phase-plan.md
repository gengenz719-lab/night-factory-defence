## 0. ゴール(何をもって完了とするか)

1. **スクショテスト合格**: スクリーンショット1枚で「走る車両の中でゾンビと戦う協力ゲーム」だと伝わる(Steamデモ合格ラインの1つ)
2. **走行感**: 「常に前進している」視覚体験(仕様13 §0の約束4)が画面から感じられる
3. **描画予算内**: Spike Cで確定した予算(レイヤー数・パーティクル量)を超えない
4. 出口成果物: **ストアページ用スクショ5枚+15秒プレイGIF**が撮れる状態

## 2. 体制と分担

| 役割 | 担当 | 内容 |
|---|---|---|
| アート生成・見た目の最終判断 | **あなた** | AI画像生成(既存パイプライン)、採用判断。いちばん楽しい部分 |
| 組み込み・シェーダー・パーティクル・UI | **Codex** | 指示書(WP形式)ベース。1 WP=1 PR |
| 規格策定・整合レビュー・予算管理 | **PM(私)** | スプライト規格、docs/16 §12予算との照合、PRレビュー |

**ルール(全WP共通):**
- 生成アートは制作方法を `godot/assets/art/README.md` に記録(Steam AI開示用)
- スプライトは **G0で定める規格書に準拠**(セル64px基準・命名・ピボット・パレット)
- 機能ロジックに触れない(見た目の差し替えとジュースのみ。装飾システムG6だけ例外)
- 各WP完了時にスモークテスト+2プロセステスト合格を維持

## 3. 作業パッケージ(優先順)

### G0: 基盤固め — 予算とルール(最初の週末)

1. **Spike C実施**(Codex、指示書発行済み)→ 背景レイヤー数・パーティクル予算・敵60体時のfpsを確定
2. **スプライト規格書**(PM作成 → docs/art/sprite-spec.md): サイズ表(キャラ/敵/モジュール/車両)、ピボット、パレット、命名規則、書き出し形式
3. **レイアウト凍結宣言**: 車両グリッド寸法・カメラ距離・HUD配置を現状で固定(変えたい点があれば凍結前に今決める)
4. パートナーテスト第1回(未実施なら並行で)

### G1: 世界の見た目 — 走行感(スクショ効果が最大)

- パララックス背景3〜5層の実アート化(空/遠景/中景/近景/路面)— 層数はSpike C結果に従う
- 路面の流れ・スピード表現(ダッシュ、飛ぶ破片、砂埃)
- 時間帯/天候のカラーグレーディング1種(ルートノードと連動できる設計にする。将来の地域差の土台)

### G2: 車両とモジュール(画面の主役)

- モジュール4種(発電機/射撃窓/タレット/工作台)のスプライト化
- 外装ダメージの段階表現(無傷→損傷→破壊=侵入口)と侵入口の見た目
- 車両の走行アニメ(車体の揺れ・車輪回転・排気)

### G3: キャラクターと敵の動き

- ガンナー/エンジニアの見た目差分(色/装備替えから開始)
- 基本モーション: 待機/走り/はしご/回避/ダウン/蘇生
  (方式はG0で決定: スプライトシート or 現行1枚絵+プロシージャル変形。コスト差大)
- 敵3種の動き・侵入モーション・攻撃予告の見た目統一

### G4: エフェクト=ジュース

- マズルフラッシュ/着弾スパーク/死亡バースト(旧Unity版v0.3の演出設計を参考にできる)
- 画面揺れ・被弾ビネット・ダメージ数字
- パーティクルはGPU、予算はSpike C内(docs/14 §12)

### G5: UI/HUDテーマ

- Theme統一(docs/14 §15: 色・フォントを散在させない)、日本語フォント確定
- ロビー/レリック投票/ルートカード/リザルトのビジュアル統一
- HUD配置はG0で凍結したレイアウトに従う

### G6: 装飾システム最小版(機能実装を伴う唯一のWP)

- 車内に「機能なし装飾」を置ける最小実装(壁紙・照明色・小物数点)
- 仕様13 §4.7「機能グリッドと装飾レイヤー分離」の土台
- スクショ映え+「自分の車両」感 → ストアページ素材に直結

### マイルストーン

```text
M-G1: G0+G1完了 → 「走ってる感」スクショをパートナー/SNSで反応確認
M-G2: G2+G4完了 → スクショテスト本番(1枚で伝わるか)
M-G3: G3+G5+G6完了 → ストアページ用スクショ5枚+GIF撮影 → フェーズ完了
```

## 4. やらないこと(このフェーズでは)

- 10Wave分の敵7種・ボス2種のアート(フルラン・アルファ側)
- 地域2種の背景差分(1種の土台まで)
- BGM・効果音の本格制作(仮SEはG4で可)
- ミニマップ、キャラカスタマイズ、装飾の品揃え拡充
- WP2-6(ゲームパッド)— 別トラックで任意実施
