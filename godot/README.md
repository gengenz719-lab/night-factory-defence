# Godot版 テストプレイ

仕様書を元にした最初のソロ縦切りプロトタイプです。完成版ではなく、車内移動・走行感・Wave戦闘・修理・レリック選択の手触りを確認するための版です。

## 起動

`PLAY_TEST.bat`をダブルクリックしてください。

この開発環境では、リポジトリ内の`.tools/godot-4.7/`に配置したGodot 4.7 stableを使用します。Godot本体はGit管理しません。

## 操作

| 操作 | キー |
|---|---|
| 左右移動 | A / D、左右キー |
| ジャンプ | Space |
| 回避 | Shift |
| はしごを上る | はしご付近でSpace / W |
| はしごを下りる | はしご付近でS |
| 照準 | マウス |
| 射撃 | 左クリック |
| 修理 / 味方蘇生 | 対象付近でE長押し |
| キャラクター固有能力 | Q |
| 車両改装を開く / 閉じる | 準備中にB |
| 準備時間を短縮 | F1 |
| Waveをテスト短縮 | F2 |
| リザルトから再開 | R |

## テストの流れ

1. F1でWave 1を開始する。
2. 車内上下階・屋根を移動しながら左右から来る敵を撃つ。
3. 外装が損傷したら、下階中央の緑色端末でEを長押しして修理する。
4. 必要ならF2で残り時間を短縮する。
5. Wave 1後、全員で同じ3枚の共有レリック候補へ投票する。ソロは選択時に即決する。
6. 続けて2つのルート候補へ投票する。安全な道路と廃品回収地で、敵構成と報酬が変わる。
7. 次Waveの準備中にBを押し、モジュールを選んで8×4グリッドへ配置する。
8. Wave 2ではランナーが外装の破損箇所を狙う。クライマー侵入は自動テストでも確認する。
9. 外装が破壊されると侵入口が開き、侵入敵が車内のクルーや設備を攻撃する。
10. 次の準備中に工作台で修理すると、8補給物資で侵入口を封鎖できる。
11. Wave 2を耐えると、選択ルートの報酬を得て仮勝利リザルトになる。

## 現在入っているもの

- 画面左向きの断面車両と、左→右へ流れる3層背景
- 下階、上階、屋根、はしご
- 射撃、敵撃破、プレイヤーHP
- 1.5セル回避、回避無敵、クールダウン
- 20秒ダウン、味方による3秒蘇生、離脱後8秒での自動復帰
- ソロ / ENetホスト / IP指定参加の最小ロビーと2人同期
- ロビーでのガンナー / エンジニア選択（重複可、ホスト確定）
- ガンナー固有能力「Combat Focus」（6秒間、攻撃速度+30% / リロード速度+50%）と武器ダメージ+10%
- エンジニア固有能力「Repair Drone」（8秒間、毎秒12HPを資材消費なしで修理）と修理速度+35% / 設備操作速度+15%
- キャラクター別の初期武器（ライフル / ヘビーリボルバー）
- 8×4・2階層の車両モジュールグリッドと準備中の配置UI
- 小型発電機、射撃窓、自動タレット、工作台
- はしご・操縦席・通路を塞ぐ配置の拒否
- 発電量と消費電力、3段階の優先度、電力不足時の低優先設備停止
- タレットの自動索敵、熱100での停止、熱50まで冷却後の復帰
- 戦闘中60% / Wave間100%の修理上限と工作台ボーナス
- 前部・後部・屋根・下部外装の破壊による侵入口生成
- ランナー / クライマーの侵入予兆と車内侵入
- 車内敵による近距離プレイヤー優先、次に稼働中モジュールへの攻撃
- Wave間の8補給物資による侵入口封鎖
- ウォーカー、ランナー、クライマー
- 前部・後部・屋根・下部外装と車体HP
- 工作台での補給物資を使った修理
- 90秒×2Wave
- 全端末共通の共有レリック3候補とチーム投票、多数決・同票時のホスト乱数確定
- シード付きRouteGeneratorによる2択ルート投票
- 安全な道路 / 廃品回収地による次Waveの敵予算・主力敵比率・報酬差分
- 2Wave突破/車両破壊リザルト

## 自動スモークテスト

```powershell
..\.tools\godot-4.7\Godot_v4.7-stable_win64_console.exe --headless --path . -- --smoke-test
```

成功時は`SMOKE_TEST_PASS`を出力します。

2人同期とバージョン拒否は次のコマンドで確認できます。

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tests\run_two_process_test.ps1
```
